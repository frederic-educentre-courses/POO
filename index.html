POO et UML

Qu'est-ce que la POO ?

La POO est un paradigme de programmation qui utilise des objets pour représenter les données et les opérations.

Class et objet

Une classe est un modèle qui définit les caractéristiques communes d'un ensemble d'objets.

Un objet est une instance d'une classe.

Propriété et méthode

Une propriété est une caractéristique d'une classe.

Ex: nom, age, sexe, etc.

Une méthode est une action que l'objet peut effectuer.

Ex: manger, dormir, etc.

Exemple de class en php


					class Personne {
						// Propriétés
						public $nom;
						public $age;
						public $sexe;

						// Constructeur
						public function __construct($nom, $age, $sexe) {
							$this->nom = $nom;
							$this->age = $age;
							$this->sexe = $sexe;
						}
					
						// Méthodes
						public function manger() {
							return "Je mange";
						}
						public function dormir() {
							return "Je dors";
						}
					}

Exemple d'objet en php


					// Création d'un objet
					$personne = new Personne("John", 30, "Masculin");
					
					// Appel des méthodes
					$personne->manger();
					$personne->dormir();

Class active ou passive

Une class active est une class qui a des méthodes qui peuvent être appelées.

Une class passive est une class qui n'a pas de méthodes qui peuvent être appelées.

Principes de la POO

  • Encapsulation
  • Héritage
  • Abstraction
  • Polymorphisme

Encapsulation

Encapsulation est une technique qui permet de cacher les données d'une classe et de les protéger.

On utilise les mots-clés private et public pour encapsuler les données.

Exemple d'encapsulation en php


					class Personne {
						// Propriétés
						public $nom;
						public $age;
						public $sexe;
						private $rib;

						// Constructeur
						public function __construct($nom, $age, $sexe, $rib) {
							$this->nom = $nom;
							$this->age = $age;
							$this->sexe = $sexe;
							$this->rib = $rib;
						}
					
						// Méthodes
						public function manger() {
							return "Je mange";
						}
						public function dormir() {
							return "Je dors";
						}
					}

Héritage

L'héritage permet à une classe d'hériter des propriétés et des méthodes d'une autre classe.

Exemple d'héritage en php


					class Personne {
						// Propriétés
						public $nom;
						public $age;
						public $sexe;

						// Constructeur
						public function __construct($nom, $age, $sexe) {
							$this->nom = $nom;
							$this->age = $age;
							$this->sexe = $sexe;
						}
					
						// Méthodes
						public function manger() {
							return "Je mange";
						}
						public function dormir() {
							return "Je dors";
						}
					}

				class Etudiant extends Personne {
					// Propriétés
					public $moyenne;

					// Constructeur
					public function __construct($nom, $age, $sexe, $moyenne) {
						parent::__construct($nom, $age, $sexe);
						$this->moyenne = $moyenne;
					}
				
					// Méthodes
					public function etudier() {
						return "Je suis en train d'étudier";
					}
				}

Abstraction

Les classes abstratites sont des classes qui ne peuvent pas être instanciées mais seulement héritées.

C'est un peu comme une extension que l'on peut utiliser quand on veut

Exemple d'abstraction en php


						abstract class Vehicule {						
							// Méthodes
							public function demarrer() {
								return "Le vehicule demarre";
							}
							public function arreter() {
								return "Le vehicule s'arrete";
							}
						}

Exemple d'utilisation de la classe abstraite


						class Voiture extends Vehicule {
							// Propriétés
							public $marque;
							public $modele;
							public $annee;

							// Constructeur
							public function __construct($marque, $modele, $annee) {
								$this->marque = $marque;
								$this->modele = $modele;
								$this->annee = $annee;
							}
						
						}

Polymorphisme

Le polymorphisme permet à une classe d'avoir plusieurs formes.

C'est à dire qu'une classe peut avoir plusieurs méthodes avec le même nom mais avec des paramètres différents.
